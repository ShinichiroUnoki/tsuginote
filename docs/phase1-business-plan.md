# Phase 1: 事業企画書

**プロジェクト**: TsugiNote（ツギノート）— AI属人化解消・ナレッジ管理SaaS
**作成日**: 2026-03-29

---

## 1. ビジネスモデルキャンバス

### 顧客セグメント
- **メイン**: 従業員10-100名の日本の中小企業（製造業、建設業、サービス業、IT企業）
- **サブ**: 士業事務所、個人事業主チーム

### 価値提案
「退職・異動しても、業務知識は残る」— AIが散在する業務知識を自動で構造化し、属人化をゼロにする

### チャネル
- コンテンツマーケティング（SEO: 「属人化 解消」「引き継ぎ マニュアル」）
- SNS（X/LinkedIn: 属人化あるある投稿）
- SaaS比較サイト（BOXIL、ITトレンド、起業LOG）
- 無料トライアル → セルフサーブ

### 顧客関係
- セルフサーブ型（低タッチ）
- ヘルプセンター + AIチャットサポート
- 月次活用レポート自動配信

### 収益の流れ
- **月額サブスクリプション**（B2B SaaS）
- 従量課金（AI生成回数のオーバーユース分）

### 主要リソース
- AIモデル（OpenAI/Anthropic API）
- RAGパイプライン（ベクトルDB + LLM）
- Webアプリケーション（FastAPI + Next.js）

### 主要活動
- プロダクト開発・改善
- AI品質チューニング（日本語精度）
- コンテンツマーケティング
- カスタマーサクセス

### パートナー
- AI API プロバイダー（OpenAI, Anthropic）
- クラウドインフラ（Vercel, Railway/Fly.io）
- SaaS比較サイト

### コスト構造
- AI API利用料（変動費: 売上の15-25%想定）
- クラウドインフラ（月額$50-200）
- ドメイン・SSL（年額$30程度）
- 開発者（自分1人: 人件費なし→生活費）

## 3. MVPスコープ

### MVP（4-6週間で開発）

**含む**:
1. ユーザー認証（メール + パスワード）
2. ワークスペース作成・管理
3. 手動ドキュメント作成・編集（Markdown）
4. AI自動ドキュメント生成（テキスト入力 → 構造化SOP）
5. ナレッジ検索（キーワード + AIセマンティック検索）
6. 引き継ぎチェックリスト（テンプレート + カスタム）
7. ダッシュボード（ナレッジ量・検索頻度）
8. 料金プラン・Stripe決済

**含まない（Phase 2以降）**:
- 音声入力 → ドキュメント変換
- ナレッジギャップ自動検出
- 日報→ナレッジ変換
- API公開
- SSO/SAML
- モバイルアプリ
- Slack/Teams連携
