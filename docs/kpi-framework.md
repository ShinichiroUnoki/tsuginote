# TsugiNote KPI体系設計

**作成日**: 2026年3月29日
**対象**: MVPローンチ〜12ヶ月目

---

## 1. North Star Metric（最重要指標）

### **Weekly Active Workspaces（WAW）— 週間アクティブワークスペース数**

| 項目 | 定義 |
|------|------|
| **定義** | 過去7日間に1回以上「AI生成」または「ナレッジ検索」を実行したワークスペース数 |
| **選定理由** | ドキュメント閲覧だけでなく、AI機能を使っているワークスペース＝本サービスの本質的価値を享受している顧客 |
| **目標** | 3ヶ月後: 15 WAW / 6ヶ月後: 50 WAW / 12ヶ月後: 150 WAW |
| **計測方法** | バックエンドのAPIログから、ワークスペースIDごとにAI生成・検索APIコール数を集計 |

**なぜWAWか:**
- ユーザー数ではなくワークスペース数 → B2B SaaSの課金単位に直結
- 単なるログインではなくAI機能利用 → 「属人化解消」という価値を実際に使っている
- 週次 → 月次より早く異変を検知でき、日次ほどノイズが多くない

## 2. AARRR Pirate Metrics

### 2-1. Acquisition（獲得）

| KPI | 定義 | 目標（月次） | 計測方法 |
|-----|------|-------------|---------|
| サイト訪問者数 | LPへのユニーク訪問者 | 3ヶ月後: 1,000 / 6ヶ月後: 5,000 | Google Analytics（またはプライバシー重視の代替: Plausible） |
| サインアップ数 | 新規アカウント作成数 | 3ヶ月後: 50 / 6ヶ月後: 200 | DBのusersテーブルのcreated_atで集計 |
| サインアップ率 | サインアップ数 / 訪問者数 | 5%以上 | 上記2指標から算出 |
| 流入チャネル別内訳 | SEO / SNS / 比較サイト / 直接 / リファラル | — | UTMパラメータで分類 |

### 2-2. Activation（活性化）

| KPI | 定義 | 目標 | 計測方法 |
|-----|------|------|---------|
| **Aha! Moment到達率** | サインアップ後7日以内にAI生成を1回以上実行した割合 | 40%以上 | ユーザーイベントログ |
| ワークスペース作成率 | サインアップ後にワークスペースを作成した割合 | 70%以上 | DBのworkspacesテーブル |
| 初回ドキュメント作成率 | サインアップ後7日以内にドキュメントを1つ以上作成 | 50%以上 | DBのdocumentsテーブル |
| メンバー招待率 | ワークスペース作成後にメンバーを1名以上招待 | 30%以上 | DBのinvitationsテーブル |

**Aha! Momentの定義:**
「初めてAI生成でドキュメントが自動作成された瞬間」＝ ユーザーが「これは使える」と感じる瞬間

### 2-3. Retention（継続）

| KPI | 定義 | 目標 | 計測方法 |
|-----|------|------|---------|
| **Week 1 Retention** | サインアップ翌週にログインしたユーザーの割合 | 50%以上 | ログインイベントログ |
| **Week 4 Retention** | サインアップ4週後にログインしたユーザーの割合 | 30%以上 | 同上 |
| **月次チャーンレート（有料）** | 当月に解約した有料顧客 / 前月末の有料顧客数 | 8%以下（6ヶ月後: 5%以下） | Stripe Subscription statusで集計 |
| **Logo Retention Rate** | 前月の有料顧客のうち今月も継続している割合 | 92%以上 | Stripeデータ |
| DAU/MAU比率 | 日次アクティブ / 月次アクティブ | 20%以上 | ログインイベントログ |

### 2-4. Revenue（収益）

| KPI | 定義 | 目標 | 計測方法 |
|-----|------|------|---------|
| **MRR（月次定期収益）** | 月末時点の有料顧客 × 各プラン月額の合計 | 3ヶ月後: ¥35K / 6ヶ月後: ¥174K / 12ヶ月後: ¥768K | Stripe MRR |
| **ARR（年間定期収益）** | MRR × 12 | 12ヶ月後: ¥9.2M | MRRから算出 |
| ARPU（顧客単価） | MRR / 有料顧客数 | ¥6,000〜¥10,000 | Stripeデータ |
| Free→有料転換率 | 有料化した顧客 / Free登録後30日経過した顧客 | 10%以上 | DB + Stripeデータ |
| Net Revenue Retention（NRR） | (前月MRR + アップグレード - ダウングレード - チャーン) / 前月MRR | 100%以上 | Stripeデータ |

### 2-5. Referral（紹介）

| KPI | 定義 | 目標 | 計測方法 |
|-----|------|------|---------|
| NPS（Net Promoter Score） | 10段階評価で9-10の割合 - 0-6の割合 | 30以上 | サービス内アンケート（四半期実施） |
| リファラル率 | 既存顧客からの紹介でサインアップした割合 | 15%以上（6ヶ月後） | リファラルコード追跡 |
| 口コミ投稿数 | BOXIL / ITトレンド等の口コミ件数 | 月3件以上 | 手動確認 |

## 3. Leading Indicators（先行指標）

将来の成長を予測する指標。悪化したら即座にアクション。

| 先行指標 | 予測対象 | 警戒閾値 | アクション |
|---------|---------|---------|----------|
| **週次AI生成回数（全体）** | MRR成長 | 前週比10%減少 | AI機能のUX改善、オンボーディング見直し |
| **Aha! Moment到達率** | Activation → Retention | 30%以下 | オンボーディングフロー改善、初回AI生成の簡略化 |
| **Week 1 Retention** | 長期Retention | 40%以下 | 初週のメール施策強化、機能案内改善 |
| **サポート問い合わせ率** | チャーン | 顧客の20%以上が問い合わせ | UX/ドキュメント改善、FAQ拡充 |
| **AI生成の「役に立った」評価率** | AI品質 → Retention | 60%以下 | プロンプトチューニング、RAG品質改善 |

## 4. Lagging Indicators（遅行指標）

結果を確認する指標。月次〜四半期で確認。

| 遅行指標 | 確認頻度 | 目標（12ヶ月後） |
|---------|---------|----------------|
| LTV（顧客生涯価値） | 四半期 | ¥300,000（推定: 平均利用25ヶ月 × ARPU ¥12,000） |
| CAC（顧客獲得コスト） | 月次 | ¥30,000以下 |
| LTV/CAC比率 | 四半期 | 10以上 |
| Payback Period | 四半期 | 3ヶ月以内 |
| 粗利率 | 月次 | 75%以上 |

## 5. ダッシュボードTop 5メトリクス

毎日確認すべき5つの数値。

| # | メトリクス | 更新頻度 | データソース |
|---|----------|---------|-------------|
| 1 | **WAW（Weekly Active Workspaces）** | 日次（直近7日ローリング） | APIログ集計 |
| 2 | **MRR** | 日次 | Stripe API |
| 3 | **新規サインアップ数**（本日/今週） | 日次 | DBクエリ |
| 4 | **Aha! Moment到達率**（直近7日） | 日次 | イベントログ集計 |
| 5 | **チャーンレート**（直近30日ローリング） | 日次 | Stripe API |

### ダッシュボード実装方針

- **MVP段階**: 管理画面にSimple KPI表示（Next.js + Stripe API + DBクエリ）
- **成長段階**: Metabase等のBIツールで可視化（PostgreSQL直接接続）
- **アラート**: 各指標の警戒閾値を超えたらSlack通知

## 6. 各KPIの計測方法とデータソース

### データソース一覧

| データソース | 取得するKPI | 取得方法 |
|------------|-----------|---------|
| **PostgreSQL（アプリDB）** | サインアップ数、ドキュメント数、ワークスペース数 | SQLクエリ |
| **APIアクセスログ（構造化JSON）** | AI生成回数、検索回数、DAU/WAU/MAU | ログ集計（定期バッチ） |
| **Stripe API** | MRR、ARR、チャーンレート、ARPU、NRR | Stripe Dashboard + API |
| **Google Analytics / Plausible** | サイト訪問者数、流入チャネル | GA/Plausible API |
| **サービス内アンケート** | NPS | 四半期実施、DB保存 |

### 集計バッチ処理

```
日次バッチ（毎朝 06:00 JST）:
  1. WAW算出 → dashboard_metricsテーブルに保存
  2. DAU算出 → 同上
  3. 新規サインアップ数集計 → 同上
  4. AI生成回数集計 → 同上
  5. 閾値チェック → 超過時Slack通知

月次バッチ（毎月1日 06:00 JST）:
  1. MRR/ARR算出（Stripe API）
  2. チャーンレート算出
  3. NRR算出
  4. Retention Cohort分析
  5. 月次レビューレポート自動生成
```

## 8. KPI段階別の重点指標

成長段階に応じて重点的に見る指標を切り替える。

| 段階 | 期間 | 重点指標 | 理由 |
|------|------|---------|------|
| **PMF探索期** | ローンチ〜3ヶ月 | Activation率、Week 1 Retention、NPS | 製品が刺さっているか確認 |
| **初期トラクション期** | 3〜6ヶ月 | WAW成長率、Free→有料転換率、CAC | 成長の再現性確認 |
| **成長期** | 6〜12ヶ月 | MRR成長率、NRR、チャーンレート | 収益の持続性確認 |
| **スケール期** | 12ヶ月〜 | LTV/CAC、粗利率、ARR | 効率的な成長確認 |

---

*本KPI体系は、MVPローンチ後の実データに基づいて四半期ごとに見直しを行うこと。特に目標値は、最初の3ヶ月間の実績をベースに再設定すること。*
