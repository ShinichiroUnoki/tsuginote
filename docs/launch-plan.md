## Day 61-90: 収益化加速

### 有料転換率の改善施策

| 施策 | 詳細 | 期待効果 |
|------|------|---------|
| **利用状況ベースのアップセル** | Free制限に近づいたユーザーに「あと5回でAI生成上限」通知 | 転換率+3% |
| **年間プラン割引** | 年払いで2ヶ月分無料（実質17%OFF） | ARPU向上、チャーン低下 |
| **Team→Businessアップグレード促進** | ナレッジ量が一定以上のチームに「ギャップ検出機能お試し」案内 | アップグレード率5% |
| **解約防止フロー** | 解約時に理由ヒアリング + 代替プラン提案 | チャーンレート-2% |

### リファラルプログラム開始

| 項目 | 内容 |
|------|------|
| **プログラム名** | 「TsugiNoteフレンド紹介」 |
| **紹介者特典** | 翌月の利用料金20%OFF |
| **被紹介者特典** | 初月50%OFF |
| **仕組み** | 紹介コード発行 → 被紹介者がサインアップ時にコード入力 → 被紹介者が有料転換後に紹介者に特典適用 |
| **目標** | 新規サインアップの15%がリファラル経由 |

### 次の機能開発の優先順位

ユーザーフィードバックに基づき、以下のフレームワークで判断:

| 優先度 | 判断基準 | 例 |
|--------|---------|------|
| **P0（即対応）** | 3社以上が同じ要望 + 有料転換/チャーン防止に直結 | Slack通知連携 |
| **P1（次スプリント）** | 有料顧客からの要望 + 差別化に寄与 | 音声入力→ドキュメント変換 |
| **P2（バックログ）** | Nice to have / 少数の要望 | モバイルアプリ |

**Growth Phase 1（3ヶ月後ローンチ想定）の候補機能**:
1. Slack/Teams連携（通知 + ナレッジ検索Bot）
2. 音声入力対応（会議議事録→ナレッジ変換）
3. ドキュメントテンプレート拡充（業種別）

## リスクと対策

| リスク | 発生確率 | 対策 |
|--------|---------|------|
| ローンチ後にクリティカルバグ発生 | 高 | ローンチ前にスモークテスト実施。Day 1-3は障害対応最優先 |
| 初期ユーザーが全く来ない | 中 | 個人ネットワーク最低20人に直接連絡。有料広告は最終手段 |
| Free止まりで有料転換しない | 中 | Week 3からプロアクティブに個別連絡。制限到達時にアップセル |
| ネガティブなフィードバック | 中 | 真摯に受け止め24時間以内に改善アクション。公開対応で信頼構築 |
