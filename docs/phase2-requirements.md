## 2. 非機能要件

### パフォーマンス

| 指標 | 目標値 |
|------|--------|
| APIレスポンス | P95 < 200ms（AI生成除く） |
| AI生成レスポンス | P95 < 10s |
| セマンティック検索 | P95 < 3s |
| ページロード（フロント） | LCP < 2.5s |

### 可用性

| 指標 | 目標値 |
|------|--------|
| アップタイム | 99.5%（月間ダウン最大3.6時間） |
| 計画停止 | 月1回以下、深夜帯 |

### セキュリティ（OWASP Top 10準拠）

| 項目 | 要件 |
|------|------|
| A01 アクセス制御 | ワークスペース間データ分離、ロールベースアクセス制御 |
| A02 暗号化 | パスワードbcryptハッシュ、通信TLS 1.3、保存時AES-256 |
| A03 インジェクション | SQLAlchemy ORM使用、入力バリデーション（Pydantic） |
| A05 設定ミス | 環境変数管理、デフォルト拒否ポリシー |
| A06 脆弱コンポーネント | Dependabot/Renovate自動更新 |
| A07 認証 | JWT + リフレッシュトークン、レート制限 |
| A09 ログ | 認証イベント・API呼び出しの構造化ログ |

### 運用要件（1人保守の制約）

| 要件 | 詳細 |
|------|------|
| 自動デプロイ | GitHub Actions → main push で本番反映 |
| 自動監視 | ヘルスチェック + エラー率アラート（Slack通知） |
| 自動バックアップ | DB日次自動バックアップ（7日保持） |
| ゼロダウンタイムデプロイ | ローリングアップデート |
| 依存関係自動更新 | Dependabot設定 |
| エラー自動通知 | Sentry → Slack連携 |
| ログ集約 | 構造化JSON ログ、一箇所で確認可能 |

### スケーラビリティ

| 項目 | 初期 | 12ヶ月後 |
|------|------|---------|
| 同時接続 | ~100 | ~500 |

## 3. ユーザーストーリー（MVP）

### 管理者として

1. ワークスペースを作成し、チームメンバーを招待したい
2. 業務マニュアルをAIに自動生成してもらいたい
3. 退職者の業務引き継ぎチェックリストを作成したい

### メンバーとして

1. 分からないことを検索し、AIに質問して即座に回答を得たい
2. 日々の業務で得た知見をドキュメントとして残したい
3. 引き継ぎ時にチェックリストに沿って進めたい

### 消費者（Free）として

1. 無料で試してみて、AIの精度を確認したい
2. チームで使う価値があるか判断したい
