## 4. 損益分岐点分析

### クラウドLLM使用時

| 項目 | 金額 |
|------|------|
| 月間固定費 | ¥25,000（インフラ¥15,000 + その他¥10,000） |
| 1顧客あたり変動費率 | 約23%（API費18% + Stripe 3.6% + その他1.4%） |
| 1顧客あたり限界利益率 | 約77% |

**損益分岐点**: 月間固定費 ¥25,000 ÷ (ARPU ¥9,600 × 限界利益率77%) = **約3.4社**

→ **有料顧客4社で黒字化**（3ヶ月目で到達見込み）

### Ollama（ローカルLLM）使用時

| 項目 | 金額 |
|------|------|
| 月間固定費 | ¥30,000（インフラ¥20,000 + GPU VPS ¥10,000 + その他） |
| 1顧客あたり変動費率 | 約5%（Stripe 3.6% + その他1.4%） |
| 1顧客あたり限界利益率 | 約95% |

**損益分岐点**: ¥30,000 ÷ (¥9,600 × 95%) = **約3.3社**

→ 損益分岐点はほぼ同じだが、顧客増加時の利益率が大幅に改善。

### マイナス月の乗り切り方

1. **月1-2のみがマイナス期間**（基本シナリオ）: 累積赤字は最大¥26,399
2. **個人の貯蓄/副業収入で補填可能な範囲**（月¥17,000 = 日額約¥567）
3. **コスト削減レバー**:
   - Free tier VPS（Railway/Render無料枠）で初期インフラ費を¥0に

   - → 最小構成では月間コスト¥2,000（ドメイン+α）まで圧縮可能

## 7. 撤退判断の定量基準

### 即時判断ポイント

| 判定時期 | 指標 | 撤退閾値 | 判断 |
|---------|------|---------|------|
| ローンチ+3ヶ月 | 有料顧客数 | 3社未満 | ピボット検討 |
| ローンチ+6ヶ月 | MRR | ¥100,000未満 | ピボットまたは撤退 |
| ローンチ+6ヶ月 | 月次チャーンレート | 15%超（2ヶ月連続） | PMF未達→ピボット |
| ローンチ+12ヶ月 | MRR | ¥500,000未満 | 撤退検討 |
| 常時 | 累積投資額 | ¥500,000超 | 投資ストップ |
| 常時 | NPS | -20以下（2四半期連続） | ピボットまたは撤退 |

### 撤退の段階

| ステージ | 条件 | アクション |
|---------|------|----------|
| **Stage 1: 警戒** | 上記閾値の80%に達した | 原因分析、改善施策を2週間で実行 |
| **Stage 2: ピボット検討** | 閾値を1ヶ月超過 | 隣接市場（カスタマーサポート、社内教育）へのピボットを検討 |
| **Stage 3: ピボット実行** | 閾値を2ヶ月連続超過 | コア技術（RAG + LLM）を活かした別ドメインに転換（2-4週間） |
| **Stage 4: 撤退** | ピボット後3ヶ月でも改善なし | グレースフル・シャットダウン（30日前通知→データエクスポート→停止） |

### 撤退時のコスト

| 項目 | 金額 |
|------|------|
| 残存インフラ費（1ヶ月分） | ¥15,000〜¥50,000 |
| ユーザーへの返金（最大） | 直近月の有料顧客分 |
| **最大撤退コスト** | **¥100,000以内**（推定） |

→ ブートストラップのため、撤退コストは極めて低い。
